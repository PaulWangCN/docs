## 简单配置应用
点击**创建应用**，输入应用名称以及应用描述，选择**简易配置**，点击**创建**，进入简易配置应用设置页面。
![](/images/build/build_1.png "")

左侧为应用信息，右侧为调试预览界面。
- **应用名称**：提问时对话框的标题和名字。
- **应用描述**：对应用场景及用途的简要描述。
- **AI模型**：可选择在**系统设置**-**模型管理**中添加的大语言模型，也可直接添加。
- **系统角色**：通过给模型指定一个特定的角色或身份，来指导模型的输出更加符合特定的场景或任务需求。
- **历史聊天记录**：大模型提交当前会话中最后 `N` 条对话内容，否则仅向大模型提交当前问题。
- **关联知识库**：用户提问优先在关联的知识库中检索分段，引用分段生成提示词发送给大模型进行询问。若未关联知识库或未匹配到分段内容，则默认将用户问题发送给大模型进行询问。知识库可设置检索方式、知识库的相似度，引用分段数 `Top-N` 和最大引用字符数、无引用知识库分段时的回答策略以及是否进行问题优化等。
- **提示词**：系统默认有智能知识库的提示词，用户可以自定义通过调整提示词内容，可以引导大模型聊天方向，该提示词会被固定在上下文的开头。可以使用变量，例如：`{data}` 是引用知识库中已知信息，`{question}`是用户提出的问题。目前可针对是否引用知识库设置不同的提示词。
- **开场白**：打开对话时，系统弹出的默认引导说明。支持 `Markdown` 格式，[-]后的内容为快捷问题，一行一个。
- **输出思考**：输出思路的一个具体过程。
- **语音输入**：在语音输入完成后会转化为文字后再发送提问，需要语音识别模型的支持。
- **语音播放**：将大模型生成的回答内容转换为语音进行播放，需要语音合成模型的支持。
应用信息设置完成后，可在右侧调试预览中进行提问测试，调试过程不计入对话日志。

点击**保存并发布**后，应用设置才生效。
![](/images/build/build_2.png "")

## 高级编排

### 串行与并行执行
流程根据节点连接方式的不同而以不同的方式执行。
当您将节点一个接一个地连接时，它们按顺序执行。每个节点都会等待前一个节点完成后再开始。每个节点都可以使用链中在它之前运行的任何节点的变量。
![](/images/build/build_3.png "")
当您将多个节点连接到同一个起始节点时，它们会同时运行。节点不能引用并行节点的输出。
![](/images/build/build_4.png "")

### 变量访问
在串行流程中，节点可以访问链中任何前序节点的变量。

在并行流程中，节点可以访问并行分割之前运行的节点的变量，但它们无法访问其他并行节点的变量，因为这些节点是同时运行的。

并行分支完成后，下游节点可以访问所有并行输出的变量。

## 使用MCP工具

### 添加MCP服务器
在您的工作空间中转到**工具** → **MCP**。
![](/images/build/build_5.png "")

输入必填参数：
- **服务端点URL**：`MCP` 服务器的位置（如 https://api.ai.qingyao.link/mcp）
- **名称和图标**：给它一个有用的名称。清奇领航会尝试自动获取图标。
- **服务器标识符**：唯一标识符（小写字母、数字、下划线、连字符，最多 `24` 个字符）
填写完成后点击保存，`MCP` 服务器添加完成。

### 接下来会发生什么
清奇领航会自动：
1. 连接到服务器
2. 处理任何 `OAuth` 认证
3. 获取可用工具列表
4. 使它们在您的应用程序构建器中可用
一旦找到工具，您将看到服务器卡片：
![](/images/build/build_6.png "")

### 管理服务器
点击任何服务器卡片可以看到该 `MCP` 支持的所有功能。
![](/images/build/build_7.png "")

点击更新按钮，可以更新该 `MCP` 支持的功能列表。
![](/images/build/build_8.png "")

点击卡片的更多操作，可以对 `MCP` 服务器的配置进行修改和删除：
![](/images/build/build_9.png "")

### 使用 MCP 工具
连接后，`MCP` 工具会出现在您期望的所有地方：

**在工作流中**：`MCP` 工具作为节点可用

**在智能代理节点中**：与常规智能代理相同

### 故障排除
- **未配置**：检查 `URL` 并重新授权
- **缺少工具**：点击”更新”
