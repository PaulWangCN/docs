清奇领航借助大语言模型显著简化了智能体的搭建过程。在充分发挥大模型优势的同时，平台还支持通过知识库、工作流等功能灵活配置智能体的应答逻辑，确保其回复符合预期，满足多样化场景需求。


- **编排能力**

清奇领航不仅提供多种大语言模型供选择，还支持多种编排模式，以适应不同复杂度的任务需求。
|功能|说明|
|---|---|
|**提示词工程**|提示词是指导模型行为与响应方式的上下文指令。您可以设定智能体的角色、明确应答范围与禁忌、定义回复格式，并提供参考信息以获得更准确的输出。清奇领航支持使用 AI 功能优化提示词，使其结构更清晰、逻辑更严谨，从而更好地引导模型生成响应。|
|**模型选择**|清奇领航开发平台提供丰富的模型资源。豆包、DeepSeek 等模型；您也可根据业务场景调整 `Temperature` 等模型参数，以优化对话效果。更多信息请参考模型管理模块。|
|**编排模式**|智能体默认采用单 `Agent` 模式，由单一智能体独立完成任务。如需构建功能更复杂、能力更全面的智能体，可选择多 `Agent` 模式，将任务拆解并由不同 `Agent` 分工协作，从而提升处理效率与系统可维护性。|

- **搭建能力**

除了依赖模型能力外，您还可通过知识库、插件等功能补充模型在专业领域知识、实时信息等方面的不足。清奇领航还提供丰富的记忆功能，使智能体能基于用户历史对话提供更个性化、准确的回复。

- **技能**

技能是智能体的核心能力扩展方式，可通过插件、工作流等方式突破模型原有能力边界。

|功能|说明|
|---|---|
|**插件**|通过 API 连接集成第三方平台与服务，极大拓展智能体能力。平台内置丰富插件可供直接调用，同时也支持创建自定义插件，将外部 API 快速集成至智能体中。例如，使用新闻插件获取实时资讯，或调用搜索工具查询网络信息。|
|**工作流**|工作流是一种用于实现复杂逻辑与多步骤任务的编排工具。通过拖拽任务节点，可设计结构清晰、执行可控的流程，提升智能体处理复杂任务的效率与稳定性。|
|**触发器**|触发器支持智能体在特定时间或事件触发时自动执行任务，适用于定时推送、事件响应等场景。|

- **知识**

知识库功能支持上传本地文档或在线数据，供智能体检索使用，从而提升回复的准确性与实用性。更多信息请参考知识库概述。

|知识库类型|说明|
|---|---|
|**文档知识库**|基于文本片段进行检索与内容召回，适用于问答、知识查询等场景。|
|**表格知识库**|支持按行索引匹配，也支持通过自然语言转 `SQL` 进行查询与计算。|

- **记忆**

为突破模型对话轮数限制，记忆功能可为智能体提供长期、可复用的信息存储与调用能力，实现更个性化的交互体验。

|功能|说明|
|---|---|
|**变量**|用于保存用户语言偏好、个人习惯等信息，使智能体能在对话中记住用户特征，提供个性化回复。|
|**数据库**|提供结构化数据的存储与管理能力，支持通过自然语言进行数据插入与查询。开发者可开启多用户模式，实现灵活的读写权限控制。更多信息请参考数据库。|
|**长期记忆**|模拟人类记忆机制，形成对用户的长期认知，从而在对话中提供更连贯、个性化的回复。更多信息请参考长期记忆。|
|**文件盒子**|提供多模态数据的合规存储与管理能力，支持用户反复调用已保存的图片、文档等文件。更多信息请参考文件盒子。|

## 工作流

工作流是由一系列可执行指令组成的集合，旨在实现特定业务逻辑或完成某项任务。它为应用或智能体的数据流动与任务处理提供了结构化的框架。工作流的核心价值在于将大模型的强大能力与具体业务逻辑紧密结合，通过系统化、流程化的方法，推动高效、可扩展的 AI 应用开发。

清奇领航提供了一个直观的可视化画布，您可以通过拖拽节点的方式快速构建工作流，并支持在画布上进行实时调试。在工作流画布中，您可以清晰地观察到数据流转的路径与任务执行的顺序。

清奇领航暂时仅支持工作流类型的编排，用于处理功能型请求，通过按顺序执行一系列节点来实现特定功能。适用于数据自动化处理场景，例如生成行业调研报告、产品解读、保险规划

- **节点**

工作流的核心构成是节点。每个节点都是一个具备特定功能的独立组件，代表一个独立的步骤或逻辑单元。节点负责处理数据、执行任务和运行算法，并且都具备明确的输入和输出端口。每个工作流默认包含一个开始节点和一个结束节点。

1. **开始节点**：作为工作流的起点，用于定义触发工作流所需的输入参数。
2. **结束节点**：用于输出工作流的最终运行结果。

通过引用节点的输出，您可以将多个节点连接起来，形成一个无缝衔接的操作链条。例如，您可以在代码节点的输入中引用大模型节点的输出，从而使代码节点能够处理大模型生成的结果。在工作流画布中，您可以看到这两个节点通过连线连接在一起。

![](/images/getting-started/introduce_6.png "")

在通过节点编排工作流时，灵活性和扩展性是实现高效设计的关键。

清奇领航工作流的各类节点（包括**开始节点**、**结束节点**、**输出节点**、**插件节点**、**子工作流节点**、**代码节点**、**SQL自定义节点**，以及**新增、查询、更新、删除数据节点**，还有**问答节点**、**批处理节点**、**循环节点**、**变量聚合节点**、**变量节点**、**选择器节点**等）均支持多种变量类型，例如：`String`（字符串）、`Integer`（整数）、`Number`（数字）、`Boolean`（布尔值）、`Object`（对象）、`File`（文件）和 `Array`（数组）等。

您可以根据实际需求灵活选用合适的数据类型，而无需进行额外的数据转换，从而显著提升工作流编排的灵活性与可扩展性。

## 变量

变量是存储信息的标记容器，因此您可以通过引用其名称来查找和使用该信息。在构建清奇领航应用时，您会遇到不同类型的变量：

- **输入**：您可以在用户输入节点为您应用的最终用户指定任意数量的输入变量来填写。

![](/images/getting-started/introduce_7.png "")
![](/images/getting-started/introduce_8.png "")

此外，用户输入节点还带有一组输入变量，您可以在流程中稍后引用。根据应用程序类型（工作流），会提供不同的变量。

![](/images/getting-started/introduce_9.png "")

|变量名|数据类型|描述|注释|
|---|---|---|---|
|`global.time`|`Number`|每次工作流执行的开始时间。||
|`global.history_context`|`Array[Object]`|对话过程中的历史上下文。||
|`global.application_id`|`String`|应用程序 ID：系统自动分配给每个应用程序的唯一标识符。此参数用于记录当前应用程序的基本信息。|此参数用于具有开发能力的用户区分和定位不同的工作流应用程序。|
|`global.chat_id`|`String`|聊天框交互会话的唯一 ID，将所有相关消息分组到同一对话中，确保 LLM 在相同的主题和上下文中继续聊天。||
|`global.user_id`|`String`|用户 ID：系统在用户使用工作流应用程序时自动分配给每个用户的唯一标识符。用于区分不同的用户。||

- **输出**

每个节点产生一个或多个输出，可以在后续节点中引用。

![](/images/getting-started/introduce_10.png "")

- **环境变量**

使用环境变量来存储您应用特定的敏感信息，如 `API` 密钥。这允许在密钥和应用本身之间进行清晰分离，因此在分享您的应用领域特定语言时，您不必承担暴露密码和密钥的风险。环境变量也是常量，无法更新。
